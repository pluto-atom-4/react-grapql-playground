UNBLOCKING REASON 1: Interactive States Foundation (#256 → #261)
──────────────────────────────────────────────────────────────
What #256 Provides:
  • Standardized hover state colors and animations
  • Focus ring styling (4.5:1 contrast, WCAG AAA)
  • CSS transition utilities (smooth 200ms transitions)
  • Accessible tab focus order conventions

What #261 Needs:
  • Cards need hover effects (table rows → card hover)
  • Cards need focus indicators (accessible navigation)
  • Cards need smooth transitions (expand/collapse)
  • Cards need proper tab order (keyboard navigation)

How They Connect:
  Cards inherit:
    .hover { color-shift, shadow-depth } → from Button hover patterns
    .focus { ring-2 ring-blue-500 } → from form input patterns
    transition-all duration-200 → from component transitions
  Result: Consistent, accessible card interactions

UNBLOCKING REASON 2: Data Flow Architecture (#295 → #261)
──────────────────────────────────────────────────────────
What #295 Provides:
  • Props-based data flow (no internal hooks)
  • Parent component fetches all data
  • Children are presentational (dumb components)
  • Error boundaries per component
  • Real-time event handling pattern

What #261 Needs:
  • Cards receive data via props (parent BuildTable passes them)
  • Cards don't fetch their own data (clean separation)
  • Error handling follows pattern (per-card error boundary)
  • Real-time updates work (Event Bus updates parent)

How They Connect:
  BuildTable (parent):
    → Fetches all build data via Apollo
    → Maps data to Card components
    → Passes: build, status, events, handlers
    → Listens to Event Bus for real-time updates
  
  BuildCard (child):
    → Receives props: build, status, events
    → Renders card UI (no data fetching)
    → Calls handlers: onEdit, onView, etc.
    → Updates on parent event changes

UNBLOCKING REASON 3: Complete UX Stack
───────────────────────────────────────
#256 + #295 together provide:

1. Polish Layer (#256)
   ✓ Hover effects, focus rings, transitions
   ✓ Users feel responsive, accessible UI

2. Architecture Layer (#295)
   ✓ Clean data flow, error handling, real-time
   ✓ Developers build reliable, maintainable UI

3. Mobile Layer (#261) Benefits
   ✓ Polish: Cards have consistent interactive states
   ✓ Architecture: Clean component composition
   ✓ Result: Responsive, accessible, maintainable cards
