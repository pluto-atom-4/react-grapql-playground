KEY RECOMMENDATIONS
================================================================================

APPROACH: Option A - Apollo RetryLink + TimeoutLink (Complex)

Why this approach wins:
- Built on existing graphql-error-handler.ts utilities
- Apollo best practice pattern
- Separates concerns (timeout vs retry vs error handling)
- Apollo DevTools compatible
- Easier to extend (circuit breakers, rate limiting)
- Interview-grade code demonstrates architectural thinking

LINK CHAIN ORDER (Critical):
  timeoutLink → retryLink → errorLink → authLink → httpLink
  
Why this order:
1. Timeout FIRST: Enforce hard 10s boundary
2. Retry SECOND: Don't waste retries on timed-out requests
3. Error THIRD: Only toast after all retries exhausted
4. Auth FOURTH: Inject token for each retry attempt
5. HTTP LAST: Actual network request
