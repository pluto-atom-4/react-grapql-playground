DEVELOPER BRANCH TRACKING (10-STEP WORKFLOW)
  1. Create branch: git branch feat/issue-#<N>-... origin/main
  2. Check current: git branch
  3. Switch branch: git switch feat/issue-#<N>-...
  4. Implement feature
  5. Track files in execution plan
  6. Stage tracked: git add [file1 file2 ...]
  7. Verify staged: git diff --cached
  8. Commit: git commit -m "feat(#<N>): ..."
  9. Push: git push -u origin feat/issue-#<N>-...
  10. Create PR: gh pr create --base main ...

REVIEWER CONSOLIDATION (FOR INTEGRATION TESTING)
  1. Create work branch: git branch work/phase-<N>-consolidation origin/main
  2. Switch: git switch work/phase-<N>-consolidation
  3. Merge features: git merge feat/issue-#35-...; git merge feat/issue-#40-...
  4. Test: pnpm test --run
  5. Review findings
  6. Delete: git branch -d work/phase-<N>-consolidation
  7. Delete remote: git push origin --delete work/phase-<N>-consolidation
  ❌ DO NOT create PR from work branch

FEATURE BRANCH CONVENTION
  feat/issue-#<issue-number>-<kebab-case-description>
  Example: feat/issue-#35-add-skeleton-loading

CONSOLIDATION BRANCH CONVENTION
  feat/phase-<number>-consolidation
  Example: feat/phase-4-consolidation
  (Recommend: work/phase-<number>-consolidation for clarity)
